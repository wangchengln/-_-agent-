Ch 3. FuFan OpenClaw 项目开发指南 二、安装部署与使用指南

安装部署 环境配置 功能使用 API 参考 项目结构

本节目标：手把手引导你完成 FuFan OpenClaw 的环境搭建、依赖安装、配置文件设置，全面介绍每个功能模块的使用方法，并提供完整的 API 接口参考和项目目录结构说明。

1. 环境要求

在开始之前，请确保你的开发环境满足以下条件：

依赖项最低版本推荐版本说明
Python3.10+3.11 / 3.12需要 | 类型联合语法和 async 增强
Node.js18.0+20 LTS / 22 LTSNext.js 14 需要 Node 18 以上
npm9.0+随 Node.js 安装或使用 yarn / pnpm 替代
Git2.30+最新版用于克隆项目仓库
操作系统Windows / macOS / Linux已在三平台测试通过
💡

API Key 准备：你至少需要一个 DeepSeek API Key（platform.deepseek.com 注册即可获取）。如需网络搜索功能，还需 Tavily API Key（tavily.com）。浏览器自动化功能需要额外安装 Playwright Chromium。

2. 安装步骤

2.1 克隆项目

git clone https://github.com/your-org/mini-OpenClaw.git
cd mini-OpenClaw

项目根目录结构如下：

mini-OpenClaw/
├── backend/          # Python 后端（FastAPI + LangChain Agent）
├── frontend/         # Next.js 14 前端
├── CLAUDE.md         # 项目指南文件
└── README.md         # 项目说明

2.2 后端安装

Step 1：创建 Python 虚拟环境（推荐）

# 创建虚拟环境
python -m venv venv

# 激活虚拟环境
# Windows:
venv\Scripts\activate
# macOS / Linux:
source venv/bin/activate

Step 2：安装 Python 依赖

cd backend
pip install -r requirements.txt

主要依赖包含：

  • fastapi + uvicorn — Web 框架与 ASGI 服务器
  • langchain + langgraph + langchain-deepseek — Agent 引擎
  • llama-index-core — RAG 检索引擎
  • sse-starlette — SSE 流式响应
  • browser-use + playwright — 浏览器自动化
  • tavily-python — 网络搜索
  • tiktoken — Token 计数
  • beautifulsoup4 + html2text — 网页内容清洗

Step 3：配置环境变量

复制 .env.example 为 .env，然后填入你的 API Key：

cp .env.example .env

编辑 .env 文件：

# === 必填 ===
DEEPSEEK_API_KEY=sk-your-deepseek-api-key
DEEPSEEK_BASE_URL=https://api.deepseek.com
DEEPSEEK_MODEL=deepseek-chat

# === 可选：OpenAI 兼容接口 ===
OPENAI_API_KEY=sk-xxx
OPENAI_BASE_URL=https://api.openai.com/v1
MODEL_NAME=gpt-4o-mini

# === 可选：Embedding 模型（RAG 模式需要）===
EMBEDDING_MODEL=text-embedding-3-small

# === 可选：Tavily 网络搜索 ===
TAVILY_API_KEY=tvly-your-tavily-api-key

# === 服务器配置 ===
HOST=0.0.0.0
PORT=8002

Step 4：安装浏览器自动化（可选）

如果需要使用 browser_use 工具，需要额外安装 Playwright Chromium：

playwright install chromium

Step 5：启动后端服务

uvicorn app:app --port 8002 --reload

启动成功后，你将看到：

🤖 Agent initialized with 7 tools (DeepSeek: deepseek-chat)
✅ fufan OpenClaw backend ready
INFO:     Uvicorn running on http://0.0.0.0:8002

访问 http://localhost:8002 可以看到健康检查返回：

{"name": "fufan OpenClaw", "version": "0.1.0", "status": "running"}

2.3 前端安装

Step 1：安装前端依赖

cd frontend
npm install

核心依赖：

  • next@^14.2.0 — React 全栈框架（App Router）
  • react@^18.3.0 — UI 框架
  • @monaco-editor/react@^4.6.0 — 代码编辑器（VS Code 内核）
  • react-markdown@^9.0.0 + remark-gfm — Markdown 渲染
  • lucide-react — 图标库
  • tailwindcss@^3.4.0 — 原子化 CSS

Step 2：启动开发服务器

npm run dev

默认在 http://localhost:3000 启动。打开浏览器即可开始使用！

✅

启动清单：确保后端（端口 8002）和前端（端口 3000）同时运行。前端通过 http://localhost:8002/api 与后端通信。

3. 配置文件详解

3.1 环境变量 (.env)

后端所有敏感配置（API Key、模型参数等）均通过 .env 文件管理，启动时由 python-dotenv 自动加载。

变量名必填说明默认值
DEEPSEEK_API_KEY是DeepSeek API 密钥—
DEEPSEEK_BASE_URL否DeepSeek API 端点https://api.deepseek.com
DEEPSEEK_MODEL否使用的模型名称deepseek-chat
OPENAI_API_KEY否OpenAI 兼容接口 Key（RAG Embedding 用）—
OPENAI_BASE_URL否OpenAI 兼容端点https://api.openai.com/v1
EMBEDDING_MODEL否Embedding 模型（RAG 模式需要）text-embedding-3-small
TAVILY_API_KEY否Tavily 网络搜索 Key—
HOST否服务器监听地址0.0.0.0
PORT否服务器端口号8002
💡

模型兼容性：由于使用了 langchain-deepseek 接口（兼容 OpenAI API 格式），你可以将 DEEPSEEK_BASE_URL 指向任何 OpenAI API 兼容的服务商（如 OpenRouter、本地 Ollama、阿里云百炼等），只需修改 URL 和 Key 即可。

3.2 Agent 人格文件 (workspace/)

workspace/ 目录下的 4 个 Markdown 文件定义了 Agent 的"人格"和"操作规范"。你可以随时修改这些文件，下一次对话就会生效。

SOUL.md — 人格灵魂

定义 Agent 的核心原则：透明优先、文件即记忆、技能即学习、安全边界。以及沟通风格：简洁、准确、有条理。

修改建议：调整 Agent 的沟通语气、增加专业领域的知识边界说明。

IDENTITY.md — 身份信息

名称、版本号、定位、语言偏好。你可以修改 Agent 的名字、风格标签和默认语言。

修改建议：自定义 Agent 名称，匹配你的团队品牌。

USER.md — 用户画像

用户的职业、技术栈、偏好。Agent 会根据此文件调整回复风格和深度。对话中 Agent 也会自动更新此文件。

修改建议：填入你的职业信息、常用技术栈、语言偏好。

AGENTS.md — 操作协议

Agent 的完整操作手册：技能调用协议（先读取后执行）、记忆协议、7 大工具使用规范、Canvas 输出规范、技能创建规范。

修改建议：添加自定义的 Agent 操作规则、工具使用约束。

3.3 运行时配置 (config.json)

config.json 存储运行时可切换的配置项，目前支持：

{
  "rag_mode": false
}
配置项类型说明
rag_modebooleanfalse = 直接拼接 MEMORY.md；true = 使用 LlamaIndex BM25+向量混合检索

RAG 模式开启后，MEMORY.md 不再直接拼入 System Prompt，而是根据用户问题进行检索，只将最相关的记忆片段注入上下文。适合记忆文件内容很多、直接拼接会超出 Token 限制的场景。

⚠️

注意：RAG 模式需要 OPENAI_API_KEY（用于 Embedding），请确保 .env 中已正确配置。可在前端设置中一键切换 RAG 模式开关。

4. 功能使用指南

4.1 智能对话

基本对话

在主页的聊天输入框中输入消息，按 Enter 发送（Shift + Enter 换行）。Agent 的回复以流式方式逐字显示。

💬

多会话管理

左侧边栏自动显示所有会话。首次对话后，AI 自动生成会话标题。支持重命名（双击）和删除。

🔗

ThoughtChain 工具可视化

当 Agent 调用工具时，界面显示可展开/收起的思考链，展示工具名称、输入参数和返回结果。

📋

原始消息面板

点击聊天头部的"查看原始消息"按钮，右侧弹出调试面板，显示完整的 System / User / Assistant / Tool 消息队列。

📊

Token 计数与压缩

实时显示当前会话和系统文件的 Token 用量。当上下文过长时，可使用"压缩"功能将历史消息归档并生成摘要。

4.2 Canvas / A2UI 可视化

输入类似以下指令，Agent 会自动生成 HTML 并在右侧 Canvas 面板中渲染：

  • "帮我创建一个温度转换器"
  • "写一个 HTML 贪吃蛇游戏"
  • "做一个数据可视化仪表盘"
  • "设计一个 Todo List 应用"

Agent 输出的 HTML 被 <openclaw-canvas> 标签包裹，后端检测到后发送 canvas SSE 事件，前端在沙箱化 iframe 中安全渲染。Canvas 面板支持新窗口打开和源代码查看。

4.3 记忆管理 (/memory)

导航到 /memory 页面，可以管理 Agent 的所有记忆文件：

文件列表

以卡片形式展示 workspace/ 和 memory/ 下的所有 Markdown 文件。点击任一卡片在右侧打开编辑器。

Monaco 编辑器

使用 VS Code 同款编辑器内核，支持 Markdown 语法高亮、自动缩进、搜索替换。编辑后点击"保存"即时生效。

AI 优化

点击"AI 优化"按钮，让 AI 帮你整理和精简记忆内容。AI 会分析当前记忆文件，删除冗余信息，重新组织结构。

实时生效

修改任何记忆文件后，下一次对话时 Agent 就会使用更新后的内容。无需重启后端服务。

4.4 技能管理 (/skills)

导航到 /skills 页面，可以管理 Agent 的所有技能：

📚

技能库浏览

以卡片列表展示所有已安装的技能。每张卡片显示技能名称、描述、启用状态。支持关键词搜索快速过滤。

✏️

在线编辑

点击技能卡片，在右侧 Monaco 编辑器中打开 SKILL.md 文件。实时编辑技能的指令内容、触发条件、执行步骤。

🤖

AI 创建技能

点击"新建技能" → 选择"AI 创建"，描述你想要的技能功能，AI 自动生成规范的 SKILL.md 文件（含 YAML frontmatter + Markdown 正文）。

📦

文件夹上传

点击"新建技能" → 选择"文件夹上传"，拖入包含 SKILL.md 的技能文件夹即可导入。支持附带 scripts/、references/、assets/ 子目录。

🏪

技能商店

浏览 GitHub 上的社区技能仓库，一键安装到本地 skills/ 目录。社区持续贡献新技能，直接复用。

4.5 设置面板

点击聊天头部的齿轮图标打开设置弹窗，支持在线管理 API Key：

  • DeepSeek API Key — 主模型接口密钥
  • OpenAI API Key — Embedding / 兼容接口密钥
  • OpenAI Base URL — 自定义 API 端点
  • Tavily API Key — 网络搜索密钥

已有 Key 以掩码形式显示（如 sk-****5435），修改后安全写入本地 .env 文件。所有密钥仅存储在本地，不会上传到任何远程服务器。

5. API 接口完整参考

后端所有 API 均挂载在 /api 前缀下。以下是完整的接口列表：

5.1 对话接口

接口说明请求体 / 参数
POST/api/chat SSE 流式对话。发送用户消息，返回 Agent 的流式响应。 {"message": "你好", "session_id": "abc123", "stream": true}

SSE 事件类型：

  • token — {"content": "..."} 逐 Token 输出
  • tool_start — {"tool": "terminal", "input": "ls"} 工具开始执行
  • tool_end — {"tool": "terminal", "output": "..."} 工具执行完毕
  • new_response — {} 工具调用后新一轮回复开始
  • retrieval — {"query": "...", "results": [...]} RAG 检索结果
  • canvas — {"html": "..."} Canvas HTML 内容
  • title — {"session_id": "...", "title": "..."} AI 生成的会话标题
  • done — {"content": "...", "session_id": "..."} 回复完成
  • error — {"error": "..."} 执行出错

5.2 会话管理接口

接口说明备注
GET/api/sessions 列出所有会话 返回 [{id, title, updated_at}, ...]
POST/api/sessions 创建新会话 返回 {id, title, created_at, updated_at}
PUT/api/sessions/{id} 重命名会话 请求体 {"title": "新标题"}
DELETE/api/sessions/{id} 删除会话 删除对应 JSON 文件
GET/api/sessions/{id}/history 获取会话消息历史 返回 messages 数组
GET/api/sessions/{id}/raw 获取原始消息数据 含 title, messages (包括 tool_calls)

5.3 文件操作接口

接口说明备注
GET/api/files?path=... 读取文件内容 返回 {"content": "...", "path": "..."}
POST/api/files 保存文件内容 请求体 {"path": "...", "content": "..."}

5.4 Token 与压缩接口

接口说明备注
GET/api/tokens/session/{id} 获取会话 Token 数 使用 tiktoken 计算
GET/api/tokens/files 获取系统文件 Token 数 统计 System Prompt 各组件的 Token 占用
POST/api/sessions/{id}/compress 压缩会话历史 将早期消息归档，生成摘要注入上下文

5.5 配置与技能接口

接口说明备注
GET/api/config 获取 RAG 模式状态 返回 {"rag_mode": false}
POST/api/config 切换 RAG 模式 请求体 {"rag_mode": true}
GET/api/config/api-keys 获取 API Key（掩码） 返回掩码后的 Key，如 "sk-****5435"
PUT/api/config/api-keys 更新 API Key 写入 .env 文件
DELETE/api/skills/{name} 删除技能 删除文件夹 + 重新生成 SKILLS_SNAPSHOT

7. 常见问题与排查

❓

Q：后端启动报错 "DEEPSEEK_API_KEY not found"

A：确保 backend/.env 文件存在且包含 DEEPSEEK_API_KEY=sk-your-key。注意 .env 文件不能有 BOM 头（建议用 VS Code 以 UTF-8 编码保存）。

❓

Q：前端连不上后端，显示网络错误

A：确认后端正在 :8002 端口运行。检查 frontend/src/lib/api.ts 中的 BASE_URL 是否指向 http://localhost:8002。确保没有防火墙或 CORS 阻止。

❓

Q：Agent 回复乱码（Windows 环境）

A：FuFan OpenClaw 已内置 UTF-8/GBK 编码安全机制（utils/encoding.py 的 safe_read_text()）。如果仍然出现编码问题，确保所有 Markdown 文件以 UTF-8 编码保存。terminal 工具已自动执行 chcp 65001 切换控制台编码。

❓

Q：browser_use 工具报错 "No chromium found"

A：需要先安装 Playwright Chromium：playwright install chromium。浏览器自动化功能依赖本地 Chromium 二进制文件。

❓

Q：RAG 模式开启后检索无结果

A：RAG 模式需要 Embedding 模型支持。确保 .env 中配置了 OPENAI_API_KEY 和 EMBEDDING_MODEL。首次开启 RAG 模式时，后端会自动构建索引（首次较慢，后续增量更新）。

❓

Q：如何更换为其他模型（GPT-4o / Claude 等）？

A：修改 .env 中的 DEEPSEEK_BASE_URL 指向对应的 API 端点，DEEPSEEK_API_KEY 填入对应 Key，DEEPSEEK_MODEL 填入模型名。例如使用 OpenRouter：DEEPSEEK_BASE_URL=https://openrouter.ai/api/v1。

❓

Q：如何添加自定义技能？

A：方法一：在 /skills 页面使用 AI 创建。方法二：手动在 backend/skills/ 下新建目录，创建 SKILL.md（含 YAML frontmatter 和指令正文），重启后端后自动加载。方法三：使用技能商店一键安装社区技能。
