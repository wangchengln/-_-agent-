1.1 Agent Skills 核心定义：从"Prompt"到"资产包"

要构建一个企业级的 Agent Skills 系统，首先必须理解 Anthropic 官方制定的标准范式。这套范式不仅仅是文件格式的约定，更是一套关于 "如何将人类专家的隐性知识（Tacit Knowledge）转化为机器可执行的显性资产（Explicit Assets）" 的工程标准。

🔗

官方资源：Agent Skills 官方介绍 — agentskills.io

在传统的 Agent 开发中，我们习惯于将所有指令都塞进 System Prompt。但在 Agent Skills 范式下，我们引入了 "技能即文件" (Skill as a File) 的核心概念。

根据 agentskills.io 的定义，Agent Skill 是一个标准化的文件夹结构，它封装了扩展智能体能力所需的特定领域知识、指令和工具定义。

它不再是一段虚无缥缈的提示词，而是一个实实在在的资产包 (Asset Package)。一个标准的 Skill 必须包含以下核心要素：

Skill = Metadata + Instructions (SOP) + Examples + Tools
🏷️

Metadata (元数据)

用于被系统索引和路由的标签（如名称、描述）。Agent 通过元数据知道"我能做什么"。

📋

Instructions (指令)

自然语言编写的算法或 SOP（标准作业程序）。这是 Skill 的"灵魂"——告诉 Agent 如何一步步执行。

📝

Examples (示例)

用于 Few-Shot Learning 的输入输出对，锁定模型行为，防止 Agent 自由发挥偏离预期。

🔧

Tools (依赖)

该技能运行所需的底层原子能力（如联网、数据库连接）。Skill 编排这些工具完成复杂任务。

这种结构化的定义，使得 Skill 具备了 可移植性 (Portability) 和 可复用性 (Reusability)。任何人都可以创建、分享、安装 Skill，就像安装 App 一样简单。

1.2 架构核心：三层渐进式加载机制

Agent Skills 系统设计的最大挑战在于 Context Window（上下文窗口）的限制。为了在有限的 Token 预算内实现无限的能力扩展，Anthropic 提出了一套 "渐进式披露" (Progressive Disclosure) 的加载机制。

这套机制将 Skill 的生命周期划分为三个层级：

Level 1 索引层 (Index Layer) — System Init

  • 加载内容：仅加载 Skill 的 元数据 (Metadata)，通常只有 name 和 description
  • 位置：常驻于 System Prompt
  • 作用：建立 "能力目录"。此时 Agent 知道自己"能做什么"（比如"能查天气"），但完全不知道"具体怎么做"
  • Token 消耗：极低（每个 Skill 约消耗 20-50 Tokens）

Level 2 指令层 (Instruction Layer) — On Demand

  • 加载内容：SKILL.md 的完整正文（包含 SOP 和 Examples）
  • 触发机制：当 Agent 的意图识别（Router）命中某个 Metadata 时，系统触发 Function Calling (read_file)
  • 位置：动态注入到 Current Context（当前对话上下文）
  • 作用：JIT (Just-in-Time) 知识注入。Agent 在这一刻才真正"学会"了该技能的业务逻辑

Level 3 执行层 (Runtime Layer) — Execution

  • 加载内容：具体的原子工具调用（Function Arguments）和外部资源
  • 触发机制：Agent 阅读指令层后，根据 SOP 发起具体的 Tool Calls
  • 作用：物理执行。调用 API、运行 Python 脚本或查询数据库，获取最终结果
💡

设计精髓：这套三层机制的核心思想是 "按需加载"。不是把所有技能的详细内容一股脑塞进上下文，而是在需要时才加载，极大地节约了 Token 预算，使得 Agent 可以挂载成百上千个技能而不爆 Context Window。

1.3 标准协议详解：SKILL.md 解剖学

在 Agent Skills 范式中，SKILL.md 是承载业务逻辑的核心载体。它采用 Markdown + YAML Frontmatter 的混合格式。一个符合规范的 SKILL.md 必须包含以下结构：

---
name: skill_name_id  # [必填] 技能唯一标识符，建议使用 snake_case
description: A concise description of what this skill does. # [必填] 用于路由匹配的关键语义信息
version: 1.0.0 # [选填] 版本控制
---

# [技能名称]

## Usage (使用场景)
明确描述在什么情况下应该激活此技能。
这有助于模型进行自我反思和意图确认。

## Steps (执行步骤)
这是 Skill 的灵魂。使用自然语言编写的算法逻辑（SOP）。
1. 第一步做什么...
2. 第二步调用哪个工具...
3. 第三步如何处理数据...

## Examples (示例)
User: [用户输入]
Assistant: [期望的思考过程和工具调用]
System: [工具返回结果]
Assistant: [最终回复]

## Considerations (注意事项)
边界条件处理、错误重试机制或安全合规要求。

YAML Frontmatter

文件顶部的 --- 包裹区域。name 用于唯一标识，description 用于 Agent 路由匹配——这是 Level 1 索引层的数据来源。

Markdown 正文

YAML 下方的正文区域。包含 Usage（何时触发）、Steps（怎么做）、Examples（示例锁定）、Considerations（边界处理）。这是 Level 2 指令层的数据来源。

1.4 关键辨析：Skill vs Tool vs MCP

在系统架构中，必须清晰界定 Skill (技能)、Tool (工具) 和 MCP (模型上下文协议) 的边界：

概念定义形象比喻职责
Skill 业务逻辑封装 (SOP) 菜谱 告诉 Agent "先切菜，再热油，最后炒"。它不直接操作物理世界，而是编排工具的使用顺序。
Tool 原子能力接口 (API) 菜刀 / 炒锅 执行具体的物理动作（如 fetch_url, execute_sql）。它没有业务逻辑，只有功能。
MCP 连接标准协议 厨房接口标准 规定了"菜刀"如何连接到"手臂"上。它解决了连接问题，但不解决"怎么做菜"的问题。

结论：Agent Skills 是位于 MCP 和 Tool 之上的 应用层 (Application Layer)。它通过编排底层的原子工具（Tool），利用标准的连接协议（MCP），来实现复杂的业务目标。

1.5 基础示例：get_weather 技能

为了验证上述理论，我们来分析一个完全符合 Anthropic 标准的最简 Skill 实现——get_weather 天气查询技能。

文件结构验证

/skills └── /get_weather <-- Skill 包（文件夹） └── SKILL.md <-- 核心定义文件

符合"技能即文件夹"的定义。下面逐层解析 SKILL.md 的各个部分：

SKILL.md 逐层解析

Level 1 Metadata（索引层）

name: get_weather
description: 获取指定城市的实时天气信息

解析：Agent 通过 description 里的语义信息，将用户查询 "北京天气怎么样" 路由到此技能。这两行只消耗约 30 Tokens，常驻于 System Prompt。

Usage（使用场景）

"当用户询问某个城市的天气情况时使用此技能。"

解析：进一步强化触发条件，帮助 Agent 在多个技能中精准匹配。

Level 2 Steps（指令层 — SOP）

从用户消息中提取城市名称
使用 fetch_url 工具访问 wttr.in/{城市}?format=j1
解析返回的 JSON 数据，提取温度、天气状况等关键信息
用友好的自然语言格式化并回复用户

解析：这是典型的 Logic Orchestration（逻辑编排）。它没有写 Python 代码，而是用自然语言指导 Agent 调用 fetch_url 这个原子工具。

Examples（对齐层）

用户：「查询北京的天气」 → Agent 调用 fetch_url → 解析 JSON → 回复：「北京当前天气：晴，温度 25°C...」

解析：通过 Few-Shot 样本锁定了 Agent 的输出格式，防止它直接把原始 JSON 扔给用户。

核心洞察：Agent Skill 的本质，就是用结构化的自然语言，编写了一段能够被 LLM 及其 Runtime 环境（Agent）所理解和执行的"软代码"。

1.6 复杂示例：docx 文档操作技能

在掌握了基础 Skill 后，我们将视线投向更复杂的业务场景。Anthropic 官方仓库中的 skills/docx 是一个教科书般的案例，它展示了 Agent 如何像一个全栈工程师一样，操控 Word 文档的创建、读取、编辑和渲染。

🔗

项目地址：github.com/anthropics/skills/tree/main/skills/docx

1.6.1 技能概览：不仅是调用，更是"全栈开发"

与 get_weather 的"单一接口调用"不同，docx 技能是一个重型资产包。它不仅教 Agent 如何使用工具，还教 Agent 如何编写代码来生成文档。

技能目标

创建专业格式的 Word 文档（含表格、目录、页眉）、提取带修订模式的内容、将文档转换为 PDF / 图片。

复杂度特征

  • 多语言混合：同时使用 Python（解包/打包）、Node.js（文档生成）、Bash（工具调用）
  • 外部依赖：集成 Pandoc、LibreOffice、docx-js
  • 状态管理：引入"解压 → 编辑 XML → 重打包"的复杂工作流

1.6.2 文件结构解剖

skills/docx/ ├── SKILL.md # 核心大脑：SOP 与知识库 └── scripts/ # 肢体动作：原子工具脚本库 └── office/ ├── unpack.py # 将 .docx 解压为 XML 目录 ├── pack.py # 将 XML 目录重打包为 .docx ├── validate.py # 验证生成的文档 └── soffice.py # 调用 LibreOffice 转换 PDF
💡

设计洞察 — "厚工具，薄指令"：为了避免在 System Prompt 中塞入冗长的 Python 代码，开发者将复杂的文件操作封装成独立的 .py 脚本（原子工具），放在 scripts/ 目录下。SKILL.md 只需告诉 Agent "运行 unpack.py"，而不需要解释"如何解压 ZIP 文件"。

1.6.3 SKILL.md 深度解析：教 Agent 写代码

docx 的 SKILL.md 最精彩的部分在于，它不仅指挥 Agent 使用工具，还指挥 Agent "现场写代码"。

A. 元数据与触发器

name: docx
description: Create, read, edit, and manipulate Word documents (.docx).
  Use when user mentions "Word doc", "report", or needs professional formatting.

B. 核心指令：双模态工作流 (Dual-Mode Workflow)

SKILL.md 中定义了两套截然不同的 SOP，分别对应"创建"和"编辑"：

模式一：从零创建 (Creation Mode)

Instructions: "To create new documents, write and execute a Node.js script using the docx library."

Agent 不再是简单地调用 create_doc(text)，而是被要求编写一段完整的 JavaScript 代码：

// Agent 学习这个模板，然后根据用户需求修改
const doc = new Document({
  sections: [{
    children: [
      new Paragraph({
        text: "Hello World",
        heading: HeadingLevel.HEADING_1
      }),
      new Table({ ... }) // Agent 需自行构建表格逻辑
    ]
  }]
});
Packer.toBuffer(doc).then(buffer =>
  fs.writeFileSync("doc.docx", buffer)
);

技术价值：这种模式赋予了 Agent 极高的灵活性。它不仅仅是在填表，而是在利用 docx-js 库的强大能力进行排版编程。

模式二：手术刀式编辑 (Surgical Editing Mode)

Instructions: "To edit existing documents with precision:"

运行 python scripts/office/unpack.py doc.docx output_dir 将 .docx 解压为原始 XML
直接读取并修改 word/document.xml 文件
运行 python scripts/office/pack.py output_dir new_doc.docx 重新打包

技术价值：这是非常硬核的操作。.docx 本质上是 XML 的压缩包。这个 Skill 教会了 Agent "外科手术"般的编辑能力——直接修改底层 XML，从而实现许多 API 无法做到的精细修改（如保留修订痕迹、修改特定样式）。

C. SKILL.md 中的关键规则

docx 的 SKILL.md 还包含了大量 docx-js 库的使用规范（快速参考表、页面尺寸、样式覆盖、表格宽度计算、图片插入、分页符等），以及编辑 XML 时的模式兼容性要求。以下是其中的核心规则摘要：

  • 显式设置页面大小 — docx-js 默认是 A4；对于美国文档，须使用 US Letter（12240 x 15840 DXA）
  • 永远不要使用 \n — 使用单独的 Paragraph 元素
  • 永远不要使用 Unicode 项符号 — 使用 LevelFormat.BULLET 和编号配置
  • 分页符必须在 Paragraph 内 — 单独使用会创建无效 XML
  • ImageRun 必须指定 type — 始终指定 png/jpg 等格式
  • 表格需要双重宽度 — columnWidths 和单元格 width 必须匹配
  • 始终使用 WidthType.DXA — 永远不用 PERCENTAGE（Google Docs 不兼容）
  • 使用 ShadingType.CLEAR — 永远不用 SOLID 作为表格阴影
  • 覆盖内置样式 — 使用准确的 ID："Heading1"、"Heading2" 等

1.6.4 原子能力层 (Atomic Tools)

为了支撑上述逻辑，Skill 挂载了几个关键的"硬原子"：

📄

Pandoc — 内容阅读

Agent 并非自己读二进制文件，而是调用 pandoc -t markdown doc.docx 将 Word 文档转译为 Markdown 文本，方便自己理解。

📦

Unpack/Pack 脚本 — 状态转换

将封闭的 .docx 转换为开放的文件夹结构（XML 文件），使 Agent 可以直接编辑底层内容。编辑后再打包回去。

🖨️

LibreOffice (Headless) — 文档渲染

当用户需要 PDF 时，Agent 调用 soffice.py 在后台启动无界面的 LibreOffice 进行转换。

1.6.5 总结：复杂 Skill 的系统论

代码生成 > 工具调用

对于复杂任务，直接让 Agent 编写并运行临时脚本（如 JS 生成文档），比预定义一千个 API 函数要灵活得多。

工具链编排

Agent 成为了一个指挥官，它指挥 Pandoc 读、Node.js 写、Python 改、LibreOffice 转，形成了一条完整的自动化流水线。

知识封装

关于"如何操作 XML"和"如何使用 docx-js 库"的知识，全部封装在 SKILL.md 中，不需要 Agent 预训练时死记硬背。

1.7 热门 Agent Skills 集成网站与开源社区

在 Agent Skills 范式下，我们不需要从零编写每一个功能。目前全球开发者已经贡献了海量的 Skill 资产，我们可以像下载 App 一样直接集成这些能力。以下是目前最热门的三个资源获取渠道：

1. Anthropic Official Skills（官方标准库）

网址：github.com/anthropics/skills

  • 官方出品：Anthropic 维护的官方仓库，代表了 Skill 开发的"最高标准"和"最佳实践"
  • 覆盖面广：涵盖从创意生成（Art/Music）、技术开发（Coding/Testing）到企业工作流（Branding/Comms）的全场景示例
  • 教学价值：每一个 Skill 都是教科书级的 SKILL.md 编写范例，非常适合新手模仿学习

2. ClawHub（OpenClaw 技能中心）

网址：clawhub.ai

  • Agent 界的"App Store"：专为 OpenClaw 生态设计的技能市场，拥有超过 50,000+ 个现成 Skill
  • 一键安装：支持通过命令行 claw install <skill-name> 直接将能力注入到 Agent 中
  • 社区驱动：拥有最活跃的第三方开发者生态，大量非官方的"黑科技"Skill 首发于此

3. Awesome Agent Skills（社区精选集）

网址：github.com/VoltAgent/awesome-agent-skills

  • 跨平台兼容：Skills 不仅适配 Claude，还兼容 Codex、Gemini CLI 等多种主流 Agent 运行时
  • 大厂集成：收录了来自 Vercel, Stripe, Cloudflare 等知名技术团队发布的官方 Skill
  • 数量庞大：目前已收录超过 300+ 高质量 Skill，是寻找灵感和"现成轮子"的宝库
🎯
