1. 立项初衷：为什么要开发 FuFan OpenClaw？

OpenClaw 是当前全球最火热的 Agent Skills 开源项目，拥有近 20 万 Star 和超过 5 万个社区 Skills。然而，当我们尝试将其应用于企业级场景时，发现了几个核心痛点：

问题一：架构过于庞大

OpenClaw 原版是一个功能完备的"AI 操作系统"，代码量巨大，依赖复杂。对于想要学习 Agent Skills 原理或进行二次开发的团队来说，上手成本极高。

问题二：黑盒化严重

虽然 Skills 本身是透明的 Markdown，但 OpenClaw 的运行时（Runtime）内部逻辑对开发者并不友好。System Prompt 如何拼接、工具如何调度、上下文如何管理——这些关键细节被封装在深层抽象中。

问题三：部署依赖重

原版依赖 Redis、PostgreSQL 等外部服务，对于个人开发者或小团队来说，部署门槛过高。

问题四：教学适配性差

作为教学项目，我们需要学生能够清晰看到每一步的执行过程——System Prompt 怎么拼的、工具怎么调的、Skills 怎么加载的——而非仅仅看到最终结果。

基于以上考量，赋范空间技术团队决定以 OpenClaw 的设计范式为蓝本，使用 LangChain 从零重新实现一版 轻量级、全透明、零外部依赖 的 Agent Skills 系统——FuFan OpenClaw。

🎯

定位：FuFan OpenClaw 不是 OpenClaw 的"阉割版"，而是一个面向教学和企业定制场景的参考实现。它完整复刻了 Agent Skills 的核心范式（指令驱动技能、动态上下文加载、热插拔架构），同时将所有运行细节完全暴露给开发者。

2. 四大设计理念

📂

文件即记忆 (File-as-Memory)

拒绝不透明的向量数据库黑盒。所有记忆以 Markdown 文件存储（SOUL.md、IDENTITY.md、USER.md、AGENTS.md、MEMORY.md），用户可以随时打开编辑。你看到的就是 Agent 的全部"大脑"。

🧩

技能即插件 (Skill-as-Plugin)

遵循 Anthropic Agent Skills 范式。技能是 Markdown 指令文件（SKILL.md），不是 Python 函数。把文件夹拖入 skills/ 目录即可扩展 Agent 能力，无需编译、无需重启。

🔍

完全透明 (Full Transparency)

System Prompt 的拼接顺序、每一次工具调用的输入输出、记忆的读写时机——所有运行细节都可以在前端的"原始消息面板"中实时查看。没有黑盒，没有魔法。

🪶

轻量化部署 (Zero External Dependencies)

不依赖 MySQL、Redis、PostgreSQL 或任何外部数据库。所有数据（会话、记忆、技能、配置）都以 JSON/Markdown 文件存储在本地文件系统。一台笔记本就能跑。

3. 技术选型

FuFan OpenClaw 的技术栈经过精心挑选，每一项选择都有明确的工程理由：

模块技术选型选型理由
后端框架Python 3.10+ / FastAPI原生 async/await 支持，SSE 流式响应性能优异

模型接口DeepSeek / OpenAI API 兼容通过 langchain_deepseek 接入，兼容 OpenRouter、Claude 等
RAG 检索LlamaIndex Core (BM25 + 向量混合)关键词 + 语义双通道检索，精度远超纯向量方案
浏览器自动化browser-use + PlaywrightAgent 可操控真实浏览器完成网页任务
前端框架Next.js 14 (App Router) / TypeScriptReact 全栈框架，SSR + 静态导出，类型安全
UI 样式Tailwind CSS + CSS 自定义属性原子化 CSS + 主题变量系统，零运行时开销
代码编辑器Monaco EditorVS Code 同款编辑器内核，记忆/技能编辑体验一流
数据存储本地文件系统 (JSON + Markdown)零依赖、完全透明、随时可用文本编辑器查看
💡

关键决策：为什么选择 create_agent 而非 AgentExecutor？

LangChain 1.x 引入的 create_agent API 基于 LangGraph 状态机，支持原生的双模式流式输出（messages + updates），可以同时获取 Token 级流式内容和工具调用的结构化事件。这是实现"完全透明"的技术基础。旧版 AgentExecutor 无法做到这一点。

4. 系统架构总览

4.1 前后端分离架构

FuFan OpenClaw 采用经典的前后端分离设计，通过 HTTP + SSE 通信：

用户浏览器 → Next.js 14 前端 (:3000) ⟶ HTTP / SSE ⟶ FastAPI 后端 (:8002) → DeepSeek LLM

前端（Next.js 14）

  • 三页面路由：/（对话）、/memory（记忆）、/skills（技能）
  • SSE 流式接收 Agent 响应
  • React Context 全局状态管理
  • Monaco Editor 在线编辑
  • 双色主题 (Bronze / Gold)

后端（FastAPI）

  • LangChain Agent 状态机引擎
  • 7 个 LangChain BaseTool 工具
  • System Prompt 动态组装
  • JSON 文件会话持久化
  • LlamaIndex RAG 混合检索

4.2 System Prompt 组装流程

这是 FuFan OpenClaw 最核心的设计之一。每次用户发送消息时，后端都会重新读取所有 Markdown 文件，按固定顺序拼接成完整的 System Prompt。这意味着你可以随时修改任何记忆文件，下一条消息就会生效。

拼接顺序（自上而下）

SKILLS_SNAPSHOT.md — 自动生成的技能清单索引（名称 + 描述 + 路径）
workspace/SOUL.md — Agent 人格灵魂：核心原则、行为边界、沟通风格
workspace/IDENTITY.md — 身份信息：名称、版本、定位、语言偏好
workspace/USER.md — 用户画像：职业、偏好、技术栈
workspace/AGENTS.md — 操作协议：技能调用流程、记忆协议、工具规范、Canvas 输出协议
memory/MEMORY.md — 跨会话长期记忆
⚠️

安全机制：任何单个组件超过 20,000 字符时，系统自动截断并附加 ...[truncated] 标记，防止 Token 爆炸。

4.3 SSE 流式事件协议

前后端通过 Server-Sent Events (SSE) 实现实时通信。后端在 Agent 执行过程中，向前端推送以下类型的事件：

事件类型说明用途
token模型生成的文字片段逐字显示 Agent 回复
tool_start开始调用工具（含工具名和参数）显示 ThoughtChain 展开动画
tool_end工具执行完毕（含返回结果）更新 ThoughtChain 结果内容
new_response工具调用后的新一轮回复开始分段显示"思考过程"与"最终回复"
retrievalRAG 检索结果显示 RetrievalCard 检索来源
canvasAgent 输出的 HTML 内容右侧 CanvasPanel 渲染预览
titleAI 为当前会话生成的标题侧边栏会话列表显示
done完整回复结束停止 loading 状态
error执行出错显示错误提示

5. 六大功能模块

5.1 智能对话系统

流式对话

基于 SSE 的逐 Token 流式输出，Agent 的每一个字都实时显示在界面上，响应延迟感知低于 200ms。

ThoughtChain 可视化

Agent 的每一次工具调用都以可展开/收起的"思考链"形式呈现。用户可以看到 Agent 调了什么工具、传了什么参数、返回了什么结果。

原始消息面板

点击"查看原始消息"，右侧弹出调试面板，完整展示 System / User / Assistant / Tool 消息队列，包括 tool_calls 的结构化数据。

会话管理

支持多会话并行、重命名、删除。会话以 JSON 文件持久化存储，结构为 {title, created_at, updated_at, messages}。

5.2 Canvas/A2UI 可视化输出

Agent-to-UI 协议

Agent 可以生成可交互的 HTML 内容（网页、图表、互动游戏、数据仪表盘等）。当 Agent 在回复中输出 <openclaw-canvas> 标签包裹的 HTML 时，后端检测并发送 canvas SSE 事件，前端在右侧 CanvasPanel 中通过沙箱化 iframe 实时渲染。

示例：输入"帮我写一个 HTML 贪吃蛇游戏"，Agent 会生成完整的 HTML 并在 Canvas 面板中呈现可玩的游戏。

5.3 五层记忆架构

🧠

SOUL.md — 人格灵魂

定义 Agent 的核心原则：透明优先、文件即记忆、技能即学习、安全边界。以及沟通风格：简洁、准确、坦诚。

🪪

IDENTITY.md — 身份信息

名称（fufan OpenClaw）、版本号、定位（本地透明AI助手）、语言偏好（默认中文）。

👤

USER.md — 用户画像

记录用户的职业、技术栈、偏好等信息。Agent 会根据用户画像调整回复风格和深度。

📋

AGENTS.md — 操作协议

Agent 的完整操作手册：技能调用协议（先读取再执行）、记忆协议（何时写入 MEMORY.md）、7 大工具使用规范、Canvas 输出规范。

💾

MEMORY.md — 长期记忆

跨会话持久化的重要信息。Agent 在对话中发现值得记住的内容时，会主动写入此文件。用户也可直接编辑。

前端提供了完整的记忆管理页面（/memory），包含 Monaco 编辑器实时编辑和 AI 优化功能（一键让 AI 帮你整理和精简记忆内容）。

5.4 技能系统

核心机制

技能 = skills/{name}/SKILL.md 文件。每个技能都是一份用自然语言编写的 SOP（操作手册），包含 YAML frontmatter（名称 + 描述）和 Markdown 正文（触发条件 + 执行步骤 + 示例）。

Agent 通过 read_file 工具读取技能文件内容，在上下文中学习步骤后，结合核心工具自主执行。

技能商店 (SkillStore)

从 GitHub 仓库浏览社区技能，一键安装到本地 skills/ 目录。

AI 创建技能

描述你想要的技能，AI 自动生成规范的 SKILL.md 文件。

文件夹上传

拖入包含 SKILL.md 的文件夹即可添加技能，支持热加载。

管理面板

搜索、启用/禁用、在线编辑、删除技能。Monaco 编辑器支持 Markdown 语法高亮。

5.6 双色主题系统

亮色主题 (Light)

Bronze 强调色 (#B8860B)，Cream 背景 (#fcfaf7)。温暖典雅，适合日间使用。

暗色主题 (Dark)

Gold 强调色 (#D4AF37)，Charcoal 背景 (#121212)。沉稳大气，适合夜间使用。

主题通过 CSS 自定义属性（--accent、--bg-surface、--text-primary 等）实现，切换时仅修改 [data-theme] 属性值，零闪烁、零重渲染。字体统一使用 Space Grotesk（via Google Fonts）。
