# 长文档生成工作流技能

## 概述
本技能提供了一个系统化的长文档生成工作流，允许用户指定主题、格式和结构，生成结构清晰、内容连贯的长文档。

## 触发条件
当用户需要生成以下类型的长文档时使用：
- 技术文档、报告、白皮书
- 项目计划书、提案
- 学术论文、研究报告
- 用户手册、教程
- 商业计划书
- 任何需要结构化格式的长篇内容

触发关键词：生成文档、创建报告、写论文、制作计划书、长文档工作流

## 执行步骤

### 第1步：需求收集
1. 询问用户文档的以下信息：
   - **主题**：文档的核心主题是什么？
   - **目的**：文档的主要用途是什么？（如：汇报、指导、分析、提案等）
   - **目标读者**：文档面向谁？（如：技术人员、管理层、普通用户、学术同行等）
   - **长度要求**：大致需要多少字或多少页？
   - **格式偏好**：是否有特定的格式要求？（如：Markdown、HTML、纯文本、特定模板）

2. 如果用户没有明确说明，提供建议选项。

### 第2步：结构设计
1. 基于收集的信息，设计文档大纲：
   - 创建层次化的章节结构（通常3-5级）
   - 确保逻辑连贯性（引言→主体→结论）
   - 考虑读者的阅读路径

2. 向用户展示建议的大纲，获取反馈和修改意见。

### 第3步：内容生成
1. 按照批准的大纲，逐章节生成内容：
   - 从引言开始，明确文档目的和范围
   - 主体部分按逻辑顺序展开
   - 确保各部分之间有平滑的过渡
   - 保持一致的术语和风格

2. 对于技术性内容，可以：
   - 使用代码块展示示例
   - 添加图表说明（用文字描述图表内容）
   - 引用相关概念时提供简要解释

### 第4步：格式化和增强
1. 根据用户选择的格式进行格式化：
   - **Markdown格式**：使用适当的标题级别、列表、代码块、表格等
   - **HTML格式**：生成结构良好的HTML文档，包含基本样式
   - **纯文本格式**：保持清晰的分段和缩进

2. 添加文档元素：
   - 标题页（包含标题、作者、日期）
   - 目录（如果文档较长）
   - 章节编号
   - 必要的脚注或尾注

### 第5步：质量检查
1. 检查文档的：
   - 结构完整性（所有章节是否完整）
   - 内容一致性（术语、风格是否统一）
   - 逻辑连贯性（论证是否合理）
   - 格式正确性（格式是否符合要求）

2. 提供摘要报告，说明文档的特点和注意事项。

## 输出格式

文档应以清晰的结构化格式输出，包含：

1. **元信息**：主题、生成日期、估计字数
2. **文档结构**：显示大纲层次
3. **完整内容**：格式化的文档正文
4. **质量摘要**：简要说明文档的特点和建议

## 示例对话

### 示例1：技术报告生成
**用户**：帮我生成一份关于"人工智能在医疗诊断中的应用"的技术报告

**Agent**：好的，我将使用长文档工作流为您生成这份报告。首先，请告诉我：
1. 这份报告的主要目的是什么？（如：技术综述、应用分析、趋势展望）
2. 目标读者是谁？（如：医疗专业人员、技术人员、管理人员）
3. 大致需要多少字？
4. 您希望用什么格式？（Markdown、HTML或其他）

**用户**：目的是技术综述，面向医疗技术人员，大约3000字，用Markdown格式

**Agent**：收到。基于您的需求，我建议以下结构：
1. 引言（AI在医疗诊断的发展背景）
2. 核心技术方法（机器学习、深度学习、图像识别等）
3. 主要应用领域（医学影像、病理分析、辅助诊断等）
4. 技术挑战与限制
5. 未来发展趋势
6. 结论

您对这个结构有什么修改意见吗？

**用户**：很好，请开始生成

**Agent**：（生成完整的Markdown格式报告...）

### 示例2：项目计划书
**用户**：我需要一个"智能家居系统开发"的项目计划书

**Agent**：我将为您创建项目计划书。请说明：
1. 计划书的详细程度？（概要版还是详细版）
2. 包含哪些部分？（如：项目概述、目标、时间线、预算、风险评估等）
3. 格式要求？
4. 是否有特定的模板需要遵循？

## 常见错误处理

1. **主题不明确**：通过提问帮助用户明确具体方向
2. **结构过于复杂**：建议简化结构，确保可读性
3. **内容重复**：检查并合并重复的部分
4. **格式问题**：根据目标格式调整样式

## 最佳实践

1. **渐进式生成**：先确认大纲，再生成内容
2. **保持一致性**：使用统一的术语和风格
3. **考虑可读性**：避免过长的段落，适当使用列表和标题
4. **提供灵活性**：允许用户在生成过程中调整需求
5. **质量优先**：宁可内容精简但质量高，不要冗长但质量低

## 注意事项

1. 对于非常长的文档（超过5000字），建议分阶段生成
2. 技术性文档需要确保术语准确
3. 商业文档需要突出关键信息和价值主张
4. 学术文档需要严谨的论证和引用
